# 70. 爬楼梯

## 题目描述

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

**示例：**
```
输入：n = 3
输出：3
解释：有三种方法可以爬到楼顶。
1. 1 阶 + 1 阶 + 1 阶
2. 1 阶 + 2 阶
3. 2 阶 + 1 阶
```

## 解题思路

### 核心：找递推关系

要到达第 n 阶，只能从两个地方来：
- 从第 n-1 阶跨 1 步
- 从第 n-2 阶跨 2 步

所以：**f(n) = f(n-1) + f(n-2)**

这就是**斐波那契数列**！

```
f(1) = 1
f(2) = 2
f(3) = f(2) + f(1) = 2 + 1 = 3
f(4) = f(3) + f(2) = 3 + 2 = 5
f(5) = f(4) + f(3) = 5 + 3 = 8
```

---

## 什么是动态规划（DP）？

**动态规划 = 把大问题拆成小问题 + 记住小问题的答案**

### 核心思想

1. **定义状态**：`dp[i]` 表示爬到第 i 阶的方法数
2. **找状态转移方程**：`dp[i] = dp[i-1] + dp[i-2]`
3. **初始化**：`dp[1] = 1`, `dp[2] = 2`
4. **计算顺序**：从小到大计算（因为大问题依赖小问题）

### 为什么不用递归？

纯递归会**重复计算**：

```
climbStairs(5)
├── climbStairs(4)
│   ├── climbStairs(3)
│   │   ├── climbStairs(2)
│   │   └── climbStairs(1)
│   └── climbStairs(2)  ← 重复计算！
└── climbStairs(3)      ← 重复计算！
    ├── climbStairs(2)  ← 重复计算！
    └── climbStairs(1)  ← 重复计算！
```

时间复杂度：O(2^n) - 指数级，n=50 就会超时

## 解法对比

### 解法1：动态规划 - 数组版（最易理解）

```cpp
int climbStairs(int n) {
    if(n == 1) return 1;
    if(n == 2) return 2;
    
    vector<int> dp(n + 1);
    dp[1] = 1;
    dp[2] = 2;
    
    for(int i = 3; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];  // 状态转移方程
    }
    
    return dp[n];
}
```

**优点：** 思路清晰，直接对应数学公式  
**缺点：** 空间复杂度 O(n)

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

---

### 解法2：动态规划 - 空间优化版（推荐）

**观察：** 计算 `dp[i]` 只需要 `dp[i-1]` 和 `dp[i-2]`，不需要保存整个数组！

```cpp
int climbStairs(int n) {
    if(n == 1) return 1;
    if(n == 2) return 2;
    
    int prev2 = 1;  // dp[i-2]
    int prev1 = 2;  // dp[i-1]
    int curr;       // dp[i]
    
    for(int i = 3; i <= n; i++) {
        curr = prev1 + prev2;  // 计算当前值
        prev2 = prev1;         // 滚动更新
        prev1 = curr;
    }
    
    return curr;
}
```

**滚动更新过程：**
```
i=3: prev2=1, prev1=2 → curr=3 → prev2=2, prev1=3
i=4: prev2=2, prev1=3 → curr=5 → prev2=3, prev1=5
i=5: prev2=3, prev1=5 → curr=8 → prev2=5, prev1=8
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1) ✅

---

### 解法3：递归 + 记忆化

用数组记住已经计算过的结果，避免重复计算：

```cpp
vector<int> memo(n + 1, -1);  // -1 表示未计算

int helper(int n) {
    if(n == 1) return 1;
    if(n == 2) return 2;
    
    if(memo[n] != -1) return memo[n];  // 已计算过，直接返回
    
    memo[n] = helper(n-1) + helper(n-2);  // 计算并记住
    return memo[n];
}
```

**时间复杂度：** O(n) - 每个子问题只计算一次  
**空间复杂度：** O(n) - 递归栈 + memo 数组

## 常见错误

### 错误1：递归调用写错

```cpp
// ❌ 错误：会无限递归
return climbStairs(n) + climbStairs(n-1);

// ✅ 正确
return climbStairs(n-1) + climbStairs(n-2);
```

### 错误2：忘记初始化

```cpp
// ❌ 错误：dp[1] 和 dp[2] 未初始化
vector<int> dp(n + 1);
for(int i = 3; i <= n; i++) {
    dp[i] = dp[i-1] + dp[i-2];
}
```

---

## 动态规划的本质

**动态规划 = 递归的逆向思维**

- **递归**：从大问题往下拆（自顶向下）
- **动态规划**：从小问题往上推（自底向上）

```
递归：f(5) → f(4) → f(3) → f(2), f(1)
DP：  f(1), f(2) → f(3) → f(4) → f(5)
```

**关键：** DP 通过"记住"小问题的答案，避免重复计算。
