# 双数组重排

## 题目描述

给定两个数组 X 和 Y：
- X 中有 m 个元素，**从小到大排列**
- Y 中有 n 个元素，**从大到小排列**

将两个数组中的元素重新排列：
- 在 X 中按**从小到大**的顺序存放**最小的 m 个元素**
- 在 Y 中按**从大到小**的顺序存放**剩下的 n 个元素**

**约束：** 尽量不要申请新的内存来存放数组中的元素（原地操作）

---

## 示例

```
输入：
X = [1, 3, 5, 7]
Y = [8, 6, 4, 2]

输出：
X = [1, 2, 3, 4]  ← 最小的4个元素
Y = [8, 7, 6, 5]  ← 最大的4个元素
```

---

## 核心思路

### 关键观察

完成重排后，应该满足：
```
X[m-1] <= Y[n-1]
```

即：**X 的最大元素** 应该 **≤** **Y 的最小元素**

### 为什么从尾部开始交换？

1. **X 的尾部** (`X[m-1]`) 是 X 中最大的元素
2. **Y 的尾部** (`Y[n-1]`) 是 Y 中最小的元素
3. 如果 `X[m-1] > Y[n-1]`，说明：
   - X 中有"太大"的元素，应该移到 Y
   - Y 中有"太小"的元素，应该移到 X
   - 交换 `X[m-1]` 和 `Y[n-1]` 能让两个数组都向目标状态靠近

4. **为什么不从头部开始？**
   - X 的头部是最小元素，肯定属于"最小的 m 个"，不需要动
   - Y 的头部是最大元素，肯定属于"最大的 n 个"，不需要动
   - 从头部开始交换会破坏已经正确的部分

---

## 解法一：交换 + 冒泡（完全原地）

```cpp
void rearrange(vector<int>& X, vector<int>& Y) {
    int m = X.size();
    int n = Y.size();
    
    // 从两个数组的尾部开始比较
    int i = m - 1;  // X的最大元素位置
    int j = n - 1;  // Y的最小元素位置
    
    while (i >= 0 && j >= 0) {
        // 如果X的最大值 > Y的最小值，说明需要交换
        if (X[i] > Y[j]) {
            swap(X[i], Y[j]);
            
            // 交换后，X[i]变小了，需要向前冒泡保持升序
            int k = i;
            while (k > 0 && X[k] < X[k-1]) {
                swap(X[k], X[k-1]);
                k--;
            }
            
            // Y[j]变大了，需要向前冒泡保持降序
            k = j;
            while (k > 0 && Y[k] > Y[k-1]) {
                swap(Y[k], Y[k-1]);
                k--;
            }
        }
        
        i--;
        j--;
    }
}
```

### 执行过程示例

```
初始：
X = [1, 3, 5, 7]
Y = [8, 6, 4, 2]

第1轮：i=3, j=3
  X[3]=7 > Y[3]=2 → swap
  X = [1, 3, 5, 2], Y = [8, 6, 4, 7]
  冒泡X: [1, 2, 3, 5]
  冒泡Y: [8, 7, 6, 4]

第2轮：i=2, j=2
  X[2]=5 > Y[2]=4 → swap
  X = [1, 2, 3, 4], Y = [8, 7, 6, 5]
  冒泡X: [1, 2, 3, 4]
  冒泡Y: [8, 7, 6, 5]

第3轮：i=1, j=1
  X[1]=2 < Y[1]=7 → 不交换

第4轮：i=0, j=0
  X[0]=1 < Y[0]=8 → 不交换

结束
```

**时间复杂度：** O(m×n)（最坏情况，每次交换都要冒泡到头）  
**空间复杂度：** O(1)（完全原地）

## 解法二：交换 + 排序（推荐）

```cpp
void rearrange(vector<int>& X, vector<int>& Y) {
    int m = X.size();
    int n = Y.size();
    
    // 从尾部开始交换所有不符合条件的元素
    int i = m - 1;
    int j = n - 1;
    
    while (i >= 0 && j >= 0 && X[i] > Y[j]) {
        swap(X[i], Y[j]);
        i--;
        j--;
    }
    
    // 重新排序恢复有序性
    sort(X.begin(), X.end());               // 升序
    sort(Y.begin(), Y.end(), greater<int>());  // 降序
}
```

### 为什么这样更优？

1. **交换阶段**：一次性找出所有需要交换的元素对
2. **排序阶段**：利用高效的排序算法（快排/归并），而不是逐个冒泡
3. **实际效率**：现代排序算法常数很小，且对"几乎有序"的数组特别快

**时间复杂度：** O((m+n)log(m+n))  
**空间复杂度：** O(log(m+n))（排序的递归栈，可以认为是原地）

## 常见错误

### ❌ 错误1：从头部开始交换

```cpp
// 错误示范
int i = 0, j = 0;
while (i < m && j < n && X[i] < Y[j]) {  // ← 方向错了
    swap(X[i], Y[j]);
    i++;
    j++;
}
```

**为什么错？**  
X[0] 是最小的元素，本来就应该在 X 里，不需要动。从头交换会破坏正确的部分。

### ❌ 错误2：交换后忘记维护有序性

```cpp
// 错误示范
while (i >= 0 && j >= 0 && X[i] > Y[j]) {
    swap(X[i], Y[j]);
    i--;
    j--;
}
// 交换后直接返回，没有排序 ← X和Y已经乱序了！
```

### ❌ 错误3：条件判断错误

```cpp
while (i >= 0 && j >= 0 && X[i] < Y[j]) {  // ← 应该是 >
    swap(X[i], Y[j]);
    ...
}
```

只有 `X[i] > Y[j]` 时才需要交换，否则说明 X 的最大值已经比 Y 的最小值小了，不需要再调整。

## 复杂度对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| 交换+冒泡 | O(m×n) | O(1) | 完全原地 | 慢 |
| 交换+排序 | O((m+n)log(m+n)) | O(log(m+n)) | 快 | 需要递归栈 |

**实际应用：** 优先选择**解法二**，除非题目严格要求 O(1) 空间

## 扩展思考

### 如果 X 和 Y 的排序方向相同怎么办？

**题目变形：** X 和 Y 都是升序

```cpp
// X和Y都升序时
void rearrange(vector<int>& X, vector<int>& Y) {
    int i = m - 1;
    int j = 0;  // ← Y的最小元素在头部
    
    while (i >= 0 && j < n && X[i] > Y[j]) {
        swap(X[i], Y[j]);
        i--;
        j++;
    }
    
    sort(X.begin(), X.end());
    sort(Y.begin(), Y.end());
}
```

## 关键要点总结

1. **从边界开始比较**：X 的最大 vs Y 的最小
2. **交换的意义**：把 X 中"太大"的元素和 Y 中"太小"的元素互换
3. **维护有序性**：交换后必须排序或冒泡
4. **终止条件**：当 `X[i] <= Y[j]` 时，说明分界线已经正确

**核心理解：** 这道题本质是在找"最小的 m 个元素"和"最大的 n 个元素"的分界线，交换是为了调整分界线，排序是为了保持有序性。
