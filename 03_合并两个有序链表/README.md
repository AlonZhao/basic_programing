# 21. 合并两个有序链表

## 题目描述

将两个升序链表合并为一个新的升序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

**示例：**
```
输入：list1 = [1,2,4], list2 = [1,3,4]
输出：[1,1,2,3,4,4]
```

## 解题思路

### 方法一：迭代法（推荐）

**核心思想：** 使用哨兵节点 + 双指针遍历

1. **哨兵节点技巧**：创建 `dummy` 节点避免处理头节点的特殊情况
2. **双指针比较**：同时遍历两个链表，每次选择较小的节点接到结果链表
3. **处理剩余部分**：循环结束后，直接连接非空链表的剩余部分

**时间复杂度：** O(m + n)  
**空间复杂度：** O(1)

#### 什么是哨兵节点（dummy node）？

哨兵节点是链表操作中的常用技巧，用来**统一处理头节点和其他节点**。

**问题：** 头节点很特殊
- 其他节点都有前驱，可以通过 `prev->next = newNode` 插入
- 头节点没有前驱，需要单独判断和处理

**解决：** 创建一个"假头节点" dummy
```
dummy -> 1 -> 1 -> 2 -> 3 -> 4 -> 4
^                                  
|                                  
哨兵节点 (不返回)                        
```

这样第一个节点也可以通过 `fake_head->next = node` 插入，和其他节点一样处理。最后返回 `dummy.next` 跳过哨兵节点即可。

### 方法二：递归法

**递归定义：** `merge(l1, l2)` 返回合并 l1 和 l2 后的头节点

**递归逻辑：**
- 终止条件：任一链表为空，返回另一个
- 递归步骤：选择较小的节点，其 next 指向剩余部分的合并结果

**时间复杂度：** O(m + n)  
**空间复杂度：** O(m + n) - 递归栈深度

## 易错点

1. **忘记处理空链表**：需要在开始时检查边界情况
2. **哨兵节点返回错误**：应返回 `dummy.next` 而非 `dummy`
3. **忘记移动 tail 指针**：每次连接节点后要更新 `tail`
