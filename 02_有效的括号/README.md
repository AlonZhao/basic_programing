# 20. 有效的括号

> LeetCode 20 · 简单

## 题目

给定只包含 `()[]{}` 的字符串 `s`,判断是否有效。有效需要满足:

- 左括号必须用相同类型的右括号闭合
- 左括号必须以**正确的顺序**闭合
- 每个右括号都有一个对应的相同类型的左括号

```
"()"      → true
"()[]{}"  → true
"(]"      → false
"([])"    → true
"([)]"    → false
```

---

## 我的尝试(卡住了)

想写一个辅助函数把左括号映射到右括号,卡在 `switch` 语法上:

```cpp
string match(string s){
    string match_s;
    switch s:                          // ← 错
    case s == '[': match_s = ']';      // ← 错
    case s == '(': match_s = ')';
    case s == '{': match_s = '}';
}
```

主函数还没动笔,这段就编译不过。

## 我犯的错

### 错误 1:C++ `switch` 的硬规则没记住

这一段一口气踩了 4 个坑。先把 `switch` 的正确形式写下来:

```cpp
switch (变量) {            // ① 必须有括号
    case 常量值:           // ② 直接写常量,不写 ==
        语句;
        break;             // ③ 不写 break 会"贯穿"到下一个 case
    default:
        ...;
}
```

对照我的代码:

| 我的写法 | 问题 | 正确写法 |
|---|---|---|
| `switch s:` | 缺括号 | `switch (s) {` |
| `case s == '[':` | case 后只能跟**常量**,不能跟布尔表达式 | `case '[':` |
| 没有 `break` | 各 case 会贯穿,后一个 case 会覆盖前一个赋值 | 每个 case 末尾加 `break;`(或直接 `return`) |
| 函数无 `return` | 声明返回 `string` 却不返回,UB | 给每个分支加 `return`,或在末尾 `return ""` |

### 错误 2:类型选错

参数写成 `string s`,但比较的是 `'['`(单字符)。这两个东西不能直接比较,而且我要处理的就是单个字符——**应该是 `char`**。

记住:`'a'`(单引号)是 `char`,`"a"`(双引号)是 `const char*`/`string`。

### 错误 3(更深的问题):写了一个不必要的辅助函数

就算 `match` 写对了,它也是**多余的**——它把"左括号 → 右括号"映射出来,但主流程其实根本不需要这个方向的映射。

正确思路下,栈里直接存"期待出现的右括号",一步到位。我下面写解法时就不会再用到 `match` 这种辅助。

**教训**:写工具函数前先想清楚主流程要什么。先有主流程的轮廓,再决定哪些步骤值得抽出去,而不是上来就写工具。

### 思路卡点:没意识到这是栈问题

更根本的是,我没看出题目要的"最后打开的必须最先关闭"就是**后进先出 (LIFO)**——LIFO 在 C++ 里就是 `std::stack`。这是一类信号词,下次见到要立刻反应过来:

| 题目里的信号 | 大概率用到 |
|---|---|
| 嵌套 / 配对 / 闭合 | 栈 |
| "最近的"、"最后的"、"撤销" | 栈 |
| "下一个更大的元素" | 单调栈 |

## 正确解法:栈,且栈里存"期待的右括号"

```cpp
#include <stack>
#include <string>
using namespace std;

class Solution {
public:
    bool isValid(string s) {
        if (s.size() % 2) return false;          // 长度奇数直接淘汰
        stack<char> st;
        for (char c : s) {
            if (c == '(') st.push(')');
            else if (c == '[') st.push(']');
            else if (c == '{') st.push('}');
            else {
                // c 是右括号
                if (st.empty() || st.top() != c) return false;
                st.pop();
            }
        }
        return st.empty();
    }
};
```

### 拿 `"([)]"` 走一遍(应该 false)

```
i=0  c='('   push ')'             栈: [ ')' ]
i=1  c='['   push ']'             栈: [ ')', ']' ]
i=2  c=')'   栈顶是 ']' ≠ ')'      return false ✅
```

### 拿 `"([])"` 走一遍(应该 true)

```
i=0  c='('   push ')'             栈: [ ')' ]
i=1  c='['   push ']'             栈: [ ')', ']' ]
i=2  c=']'   栈顶 ']' == ']' pop  栈: [ ')' ]
i=3  c=')'   栈顶 ')' == ')' pop  栈: []
栈空 → return true ✅
```

### 三个关键点

**① 栈里存"期待的右括号",不是"读到的左括号"**

这是核心妙招。读到 `(` 时,我们已经知道"以后必须出现一个 `)`"——直接把 `)` 押进栈。这样:

- 不需要写"左→右"或"右→左"的映射函数
- 不需要 `unordered_map`
- 后面遇到右括号,栈顶就是它该长的样子,直接相等比较

这就是为什么我之前写的 `match` 函数完全不需要。

**② 三个失败条件,一行拦截**

```cpp
if (st.empty() || st.top() != c) return false;
```

同时处理:

- 字符串以右括号开头(栈空就来了右括号 → `st.empty()` 真)
- 类型对不上(栈顶不是当前字符 → `st.top() != c` 真)

`||` 的**短路求值**保证空栈时不会去访问 `st.top()`(`top()` 在空栈上是 UB)。所以 `st.empty()` 必须**写在前面**。

**③ 最后必须 `return st.empty()`,不能 `return true`**

考虑 `"(("`:循环里没触发任何错误,但扫完后栈里还剩两个 `)`,说明左括号没闭合。

**④ 长度奇数提前淘汰**

`if (s.size() % 2) return false;`——三种括号都成对出现,合法字符串长度必然是偶数。这一行能省掉整个循环,体现"先看全局约束再动手"的习惯。

## 收获清单

- [ ] C++ `switch`:`switch (x)` 要括号;`case 常量:` 不能写表达式;每个 case 要 `break` 或 `return`;函数所有路径都要 `return`。
- [ ] `'a'`(char) ≠ `"a"`(string)。处理单字符就用 `char`。
- [ ] 写工具函数前先把主流程画出来——别上来就写助手。
- [ ] **后进先出**(LIFO)= `std::stack`。看到"嵌套/配对/最近"的信号词,先想栈。
- [ ] 用 `||` 短路求值保护 `top()`、`front()` 这些不能在空容器上调的方法。
- [ ] **栈里存什么** 是个设计选择——存"期待的下一步"往往比存"读到的当前值"更省事。这是和上题 `unordered_map` 的 key/value 选择一脉相承的设计原则。
